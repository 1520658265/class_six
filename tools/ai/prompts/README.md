# AI Prompt 目录规范

本目录存放生成美术资产用的 prompt。为方便和 `docs/design/美术资产清单-spec.md` 逐项对应，新增 prompt 按资产类型分类，并使用中文文件名。

## 分类

```text
角色精灵/   角色概念图、行走表、表情、动作
场景建筑/   教学楼、宿舍、小卖部、食堂、厕所等建筑外观或内景
地表自然/   地表、道路、山路、树林、石头、水池等 tileset
校园道具/   校园杂物、小卖部商品、课桌椅、蒸笼、饭盒等
战斗武器/   书包、扫帚、饭盒、悠悠球、黑板擦等可替换武器层
战斗特效/   推搡灰尘、挥击弧线、受击星星、勇气爆发等
参考临时/   风格头、综合色板、一次性实验 prompt
```

## 命名

```text
<资产名>_<用途或规格>_v<版本>.txt
```

示例：

```text
角色精灵/主角元生_行走表_v1.txt
角色精灵/主角元生_酷帅杀马特概念_v1.txt
角色精灵/班主任鲍先进_概念_v1.txt
场景建筑/教学楼_完整10x3_v1.txt
地表自然/地表道路_tileset_v2.txt
校园道具/校园杂物_tileset_v1.txt
```

## Prompt 写作规范

每个 prompt 必须包含以下结构，缺一不可：

### 1. 风格声明

开头引用 `参考临时/通用像素风格头.txt` 的 STYLE 段（32-bit JRPG，PS1/GBA-to-DS 级别），或将其内容直接嵌入。不要只写”像素风”三个字。

### 1.5 用途声明（每个 sprite sheet 必写在 prompt 顶部）

sprite sheet 类资产的用途是**被脚本按 N 行 M 列均分裁切成独立的单元贴图**导入引擎使用。这一点必须在 prompt 开头就明说，否则 AI 会按"展示图"思路构图，导致大型元素（光晕、扩散粒子）越界吃到隔壁格。

每个 sprite sheet prompt 必须在风格声明之后、内容描述之前包含：

> 这张图会被按 N 行 × M 列均匀切割，每个格子被切出来后单独作为一张贴图使用。每个元素必须**完整落在自己那一格的方形区域内**，元素的最外缘像素距离格子边界至少 25% 安全距离。**不允许任何光晕、粒子、碎屑、阴影、描边超出自己那格**。相邻格子之间必须能看到清晰的纯洋红空白带，肉眼一眼能看出格子边界在哪。

**Why this matters**：写"占该格 70%"只管大小不管边界纪律。对硬边小物件（瓶子、箭头、雪花）侥幸够用，对漫反射光晕（暗红光柱、金光、火星）就会越界——切片时第一格切出来缺一截、第二格切出来又混进隔壁的内容。

### 2. 画布布局（关键约束）

- **默认 tile 像素规格：每个格子是 64×64 像素**（之前是 32×32，已升级以利用更高细节）。这只是 prompt 里描述用的逻辑像素规格——实际生成画布由 `--size 1K` 控制（默认）。
- **不要写画布的绝对像素尺寸**（如"宽 288 像素"）。Gemini 模型有固定输出分辨率（1K=1024 或 2K=2048），prompt 里写画布像素总数会被忽略。但**单格 tile 的像素规格 64×64 可以写**，作为 chunky pixel 风格锚定。
- **改用比例语言**：
  - "每个格子是完美正方形 1:1 比例"——保证 cell 不被压扁
  - "整张画布宽高比约 N:M"——给画布形状一个相对锚点
- **优先选择接近正方形的网格布局**（如 2×4、3×3、4×4）。Gemini 输出趋向正方形画布，强行要求 7:1、5:1 这种极端宽高比会被忽略，导致格子被压扁。
- **挑刚好合身的网格，不要凑数留空**：6 个资产用 2×3 或 3×2，4 个用 2×2，9 个用 3×3。除非该资产组本来就需要"空缺"作为占位（如未实现状态），否则不要让 prompt 出现"留空一格"的指令。
- **画布外圈不要加边框**：每格自身就是独立单元，不需要外框包住整张图。
- **每个 tile 不要画黑色或灰色描边边框**（明确禁止"frame around each cell"）。AI 看到"网格分隔"会自作主张给每格加描边，必须显式禁止。
- 通过脚本参数控制画布形状和分辨率：
  - `--aspect`：理论上控制画布比例（实测 Gemini 不一定生效，但写上无害）
  - `--size 1K`：默认 1K 分辨率（1024）。1K 配合 64×64 tile 规格，2×3 网格每格 ≈ 340 屏幕像素，5× 超采样足够清晰
  - 仅在大型 sprite sheet（如 4×4 = 16 格以上）时再升到 `--size 2K`，否则单格分辨率会过低

### 3. 间隔与背景

**核心原则**：不要让 AI 觉得”格子之间需要分隔”——它会自作主张加边框、阴影、立体效果。

正确写法：
- 描述成”整张画布是 100% 纯洋红 #FF00FF，图标按虚拟网格位置摆放在洋红海里”。
- 强调”虚拟网格不画出来，只用来定位”。
- 强调”图标之间没有任何线条或颜色块把它们分开”。
- 让相邻图标之间留出充足洋红间距，自然就独立了。

错误写法（会触发边框副作用）：
- “格子之间用洋红竖条分隔”——AI 会把这理解成需要画分隔条
- “每格用洋红边框包围”——AI 会画边框
- “tile-based grid layout”——AI 会画 tile 边线

明令禁止：
- 不要任何分隔线、分隔条、网格线、tile 描边框、外框
- 不要给整张画布加外边框
- 背景必须是 100% 连续的纯洋红色块

### 4. 每格内容描述（具象化）

- 用”想象……的瞬间”开头，给 AI 一个现实场景锚定。
- 给出具体的元素数量（”4-5 条线”、”15-20 颗粒子”）。
- 给出每个元素的 hex 色值（如 #8c7a62），不要只说”灰褐色”。
- **元素大小用”安全居中”语言而不是”占格 70%”**：
  - 错：约占其位置正方形的 70%
  - 对：元素居中放置在自己那格内，元素的所有可见部分（包括光晕、粒子、阴影、描边）都必须紧凑收拢在格子中央 50% 的内圈区域，最外缘距离格子边界至少 25% 安全距离
- 描述整体形状（”横向拉长”、”放射状星芒”、”扁平椭圆”）。
- 每格末尾加”不要画成 X、不要画成 Y”的反例锚定。
- 对漫反射光晕/粒子类元素，必须额外强调”光晕的外层渐变必须在 50% 内圈区域内完成消散，不允许向外扩散到格子边界附近”。

### 5. 颜色防混淆

洋红 #FF00FF 背景上画白色/浅色元素时，必须额外强调：
- "颜色必须是纯白 #ffffff 或极浅灰 #f0f0e8"
- "绝对不能是粉色、红色、洋红色"
- 否则 AI 会把白色偏移成粉红色，和背景混在一起。

### 6. 空白格/留空指令

如果某格要求空白，必须用最强硬的措辞：
- "这一格必须完全空白"
- "只有纯洋红 #FF00FF 填充"
- "没有任何图形、点、线、十字、占位符"
- "绝对什么都不画"

只写一句"留空"是不够的，AI 会自作主张填内容。

### 7. 禁止项（全局）

每个 prompt 末尾必须有完整的禁止清单，覆盖：
- 不要的视觉风格（魔法、武侠、科幻等）
- **不要文字、字母、数字、标签、图例、编号——画布的任何位置（包括顶部/底部/侧边）都不允许出现**。AI 经常自作主张在画布底部加英文图例行，必须用最强硬措辞禁止。
- 不要的元素（水印、签名）
- 不要的边框（**不要给每个 tile 加灰色/黑色描边框、不要给整张画布加外框**）
- 不要的背景色（白底、灰底）
- 不要的后处理效果（anti-aliasing、模糊渐变）
- 不要的布局错误（"不要生成超过 N 行"、"格子绝对不能压扁"）

### 8. 审图检查清单

**重要事实**：AI 助手没有视觉验图能力（Read PNG/JPG 返回空），不能替用户判断图片对不对。生成完成后必须**由用户验收**，AI 助手要做的是为用户提供客观的验收辅助信息：

- 用 `tools/ai/check_grid.py` 在原图上叠加 N×M 等分线（绿色），方便用户一眼看出有没有越界
- 把图片路径明确告诉用户
- 跟用户说"图已生成，我无法验图，请你检查文字/越界/比例"
- **绝对不要在用户没确认前写"图已通过""6 个特效全部正确"等断言**

用户的验收清单（供用户参考）：
- 边缘扫一圈：底部/侧边有没有混入英文字母、图例、标注？
- 每格内容是否符合预期？
- 背景是否 100% 纯洋红，无分隔线、无 tile 边框？
- 浅色元素是否变成了粉色（白光偏粉、星星偏粉）？
- 格子比例是否被压扁？
- **切片模拟验收（最关键）**：用 PIL 在原图上画一组 N×M 等分网格线叠加，检查每个元素是否完整落在自己那格内，是否有任何光晕/粒子/阴影越过分割线吃到隔壁格。

切片模拟代码（`tools/ai/check_grid.py` 已实现）：

```bash
python tools/ai/check_grid.py <sheet.png> <rows> <cols>
# 输出 <sheet>_grid.png，带绿色等分线
```

## 约定

- prompt 文件名用中文，方便人工查找和剧情/资产清单对应。
- 生成出的正式工程资源建议继续使用英文资产 ID，例如 `char_yuansheng_walk_sheet.png`。
- 旧英文 prompt 保留为历史版本，不强制删除；新 prompt 优先写入中文分类目录。
- 同一资产不同用途拆成多个 prompt，例如”概念图”和”行走表”不要混写。
- 战斗角色遵循”**身体动作和武器分层**”原则：角色本体不把武器画死在帧里，武器单独生成，再通过挂点贴到手部。
- 生成后的 AI 图片原图只做分类归档和命名整理；除非用户单独要求，不再切图、缩放、转 PNG、抠透明或做任何会改变图片本身的处理。
