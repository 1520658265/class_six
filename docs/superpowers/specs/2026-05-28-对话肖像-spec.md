# 对话肖像系统设计 spec

> 创建日期：2026-05-28
> 关联文档：[美术资产清单-spec.md](../../design/美术资产清单-spec.md) 第六节
> 关联实施：本 spec 通过后由 `superpowers:writing-plans` 出实施计划

## 一、目标

为《元生的六年级》游戏的对话系统提供 **11 个核心角色、共 40 张** 半身像贴图，作为对白演出的视觉核心资产。

## 二、规格定义

| 项 | 值 |
|------|------|
| 单 tile 像素规格 | 64×64 |
| 生成尺寸 | 命令参数使用 `--size 1K`；prompt 内不写画布绝对像素尺寸 |
| 构图 | 半身像（头+肩+少量胸口）|
| 背景 | 纯洋红 #FF00FF（切片后由引擎/工具转 alpha）|
| 风格基线 | 32-bit JRPG / PS1-GBA-DS（引用 `tools/ai/prompts/参考临时/通用像素风格头.txt`）|
| 表情风格 | 克制写实——微动作、不夸张：嘴角±2-3 像素、腮部色块表脸红、眼睛半闭表低头 |
| 不允许 | 大汗珠、椭圆"○○"惊讶眼、波浪嘴；魔法符号；姓名标签；UI 框 |

### 画布布局规则

- 每个角色一张 sprite sheet
- 网格优先选择接近正方形、便于人工验收的布局；3/5 表情允许最后 1 格强制空白：
  - 6 表情 → 2×3
  - 5 表情 → 2×3（最后留 1 空格）
  - 4 表情 → 2×2
  - 3 表情 → 2×2（最后留 1 空格）
  - 2 表情 → 1×2
- **必须按隐形等分线写 prompt**：2×3 时垂直等分线在整图宽度的 1/3、2/3，水平等分线在整图高度的 1/2；2×2 时垂直/水平等分线都在 1/2；1×2 时垂直等分线在 1/2。
- **绿色验收线只能穿过洋红背景**：prompt 必须明确“如果画绿色等分线检查，绿色线必须只穿过纯洋红 #FF00FF 背景，绝对不能切到人物的脸、头发、肩膀、衣服、手、烟雾或描边”。
- **肖像必须小而集中**：每个肖像放在自己格子的正中央，所有可见像素收在本格中央 50% 内圈区域内，并距离本格边界至少 25% 安全距离。宁可多留洋红空白，也不要贴近等分线。
- 生成命令必须传入匹配网格的 `--aspect`：
  - 2×3 → `--aspect 3:2`
  - 2×2 → `--aspect 1:1`
  - 1×2 → `--aspect 2:1`
- prompt 套用 `tools/ai/prompts/README.md` 当前规范（切片自包含、安全边距 25%、无 tile 边框、无文字、纯洋红背景）；空白格必须按 README 的“空白格/留空指令”强约束写法描述

## 五、Prompt 模板（每角色复用）

每个 prompt 必须包含以下结构（按 `tools/ai/prompts/README.md` 第 1-7 节）：

1. **风格声明**：32-bit JRPG / chunky pixel / 1px 描边
2. **用途声明：必须严格按格切片**：这张图会被按 N 行 × M 列均分裁切，每格独立成贴图；必须写清楚隐形等分线位置，并声明绿色等分线只能穿过纯洋红背景，不能切到人物
3. **画布布局**：每格逻辑规格 64×64，正方形 1:1，整图宽高比约 M:N，连续纯洋红背景；不要写画布绝对像素尺寸；虚拟网格不画出来
4. **角色外观锁定**：从上方“概念图参考”路径提取的特征（hex 色值、头型、衣领、标志物）
5. **每格摆放位置**：明确写“第 X 行第 Y 列：<表情>，肖像中心在该格中心”，避免模型把人物整体居中成群像或跨格排列
6. **每个表情格的具象描述**：嘴角动作、眼神方向、腮红、其他细微动作（如鲍先进抽烟帧的烟雾走向）
7. **强禁项**：
   - 不允许文字、字母、数字、姓名标签、图例
   - 不允许 tile 描边框、画布外框
   - 不允许夸张漫画表情（大汗珠、○○眼、波浪嘴）
   - 不允许任何元素超出自己那格
   - 不允许任何元素靠近或跨过隐形等分线
   - 不允许人物变大、居中成一整张群像、跨格排列或贴近画布边缘
   - 不允许 anti-aliasing

## 六、工作流（每角色一轮）

1. 我写 prompt（套规范模板 + 该角色已有概念图视觉锚点）
2. 用户过目 prompt，确认无问题
3. `python tools/ai/gen_with_gemini.py <prompt_file> -o tools/ai/out/角色肖像/char_<role_id>_portrait_sheet_v<n>_raw --aspect <M:N> --size 1K`
4. `python tools/ai/check_grid.py <output> N M` 输出 `<output>_grid.png` 叠绿色等分线辅助验收
5. **由用户验收**（AI 助手没有视觉验图能力，不能替用户判断）。验收清单：
   - 绿色等分线是否只穿过洋红背景（任意线切到脸/头发/肩膀/衣服/手/烟雾都判失败）
   - 角色一致性（这是不是设定里的人）
   - 表情准确度（克制写实 vs 夸张漫画）
   - 切片自包含（无元素跨格）
   - 边缘干净（无文字、无图例）
6. 通过则转为 PNG 并归档到 `assets/art/characters/<role_id>/portrait/`，失败则按用户反馈迭代 prompt

### 已验证经验（2026-05-28）

- 弱约束 prompt 即使输出尺寸接近目标比例，也可能把人物画到等分线上，导致 `check_grid.py` 的绿色线切到脸。此类结果不能验收。
- 元生 v1 失败原因：只写“2×3 虚拟网格”和安全边距不够，模型仍把肖像放大并靠近等分线。
- 元生 v2 通过经验：必须同时满足三点：prompt 明写隐形等分线位置、声明绿色线只能穿洋红背景、生成命令传 `--aspect 3:2`。
- 后续所有新角色肖像都必须以 v2 规则为基线；不要复用 v1 弱规则。

## 七、范围与排除

本 spec **包含**：
- 11 角色对话肖像 sprite sheet 的生成 prompt 设计
- 每张 sheet 的网格选择、表情清单、视觉锚点
- 命名规范与归档路径

本 spec **不包含**：
- 切片为单帧贴图的工程脚本（属于后续 import-time 工具）
- Godot 引擎中肖像 UI 的对话框集成
- 表情动画过渡（如"普通→脸红"的过渡帧）
- 配音、字幕样式

## 八、待用户确认的问题（无）

本 spec 通过四轮 brainstorming 已确认所有关键决策（构图、布局、背景、表情幅度），无遗留问题。
