## 目标

根据用户对童年母校"丁埠小学"的描述，生成一份俯视/正面混合视角的 RPG 风格地图，输出两份产物：

1. `school.png` - 渲染好的地图位图（用于预览/分享）
2. `school.tscn` - Godot 4 场景文件，含一个 TileMap 节点（用于后续在 Godot 中打开）

附带一份支持文件：

3. `school_tileset.tres` - Godot 4 TileSet 资源，被 `.tscn` 引用

**本次范围内不做：** 玩家角色、移动逻辑、碰撞体、相机、project.godot 工程文件、室内场景。仅静态地图。

## 地图布局

学校像是从一座山中间挖平建造的，**没有围墙**，四周由树林和岩石环绕。

### 山林边界规则

- 北边界：y = 0..2
- 南边界：y = 37..39
- 西边界：x = 0..2
- 东边界：x = 61..63
- 树和岩石按 70% 树 / 30% 岩石的比例随机交错放置（用确定性随机种子，保证可复现）
- 边界与建筑物之间是草地

### 地表分层（自底向上）

1. **底层**：草地（默认填充全图）
2. **道路层**：主马路 + 教学楼右侧小路（泥地 tile）
3. **建筑层**：各建筑的屋顶/墙体 tile
4. **装饰层**：树、岩石、淘米池水

## 美术风格规范（全局）

整张地图统一一套风格，所有 tile 必须遵守。

**风格基调：** 16-bit JRPG 顶视/正面混合像素画风（参考 Kenney Top-Down RPG Pack 的卡通块面感觉）+ 1990 年代中国农村小学怀旧褪色调（参考《Eastward》/《雨纪》）。

**像素规范：**
- chunky pixel：每个"逻辑像素"是清晰可见的色块，禁止 anti-aliasing、禁止平滑渐变
- 1px 深色描边（深炭灰 `#2a2520` 或深棕 `#3a2820`）只用于独立物件（门、窗、招牌、围栏、家具、自然物）
- 可平铺底材（草地、屋顶水泥面、纯瓷砖墙等）不加外描边，确保无缝拼接
- 阴影只用块状色块 + 1px dithering，光源统一为左上 45°

**配色锁定（hex）：**

| 用途 | 主色 | 高光/阴影 |
|------|------|----------|
| 屋顶水泥面 | `#8a8c85` | — |
| 屋顶女儿墙 | `#a8aaa2` | HL `#c8cac0` / SD `#6a6c65` |
| 教学楼瓷砖 | `#c89968` | 勾缝 `#b8b0a0` |
| 招牌朱红字 | `#b03828` | 描边 `#6a1f15` |
| 招牌底板 | `#ece4d0` | 1px 暗红描边 |
| 木门 | `#8a5028` | 把手 `#3a3530` |
| 窗框 | `#d8d0c0` | — |
| 窗玻璃 | `#a8c8d8` | — |
| 建筑勒脚 | `#5a5852` | — |
| 草地 | `#6a8848` | 草尖 `#4a6830` |

其它建筑（宿舍/食堂/小卖部/厕所）的坡屋顶、红砖墙等待后续 tileset 设计时再补充配色。

**禁止项：** 鲜艳荧光色、smooth gradient、AA、抖动以外的渐变、tile 内带阴影投到 tile 外（破坏拼接）。

## 资源选用：自生成像素艺术 tileset

**前期方案（已废弃）：** 直接使用 OpenGameArt 的 KenneyRPGpack 欧式中世纪 tileset。问题是风格与 1990 年代中国农村小学完全不契合（坡屋顶民居、针叶林、欧式石墙）。

**当前方案：** 通过 Gemini 图像生成模型自制专属 tileset，每个分类一张 sprite sheet，洋红色 `#FF00FF` 背景便于 chroma-key 抠图。

### 生成管线

**脚本：** `tools/ai/gen_with_gemini.py`

**最简职责（已精简）：** 读 prompt 文件 → POST 给 `bobdong.cn` 代理的 `gemini-3.1-flash-image-preview` → 保存返回的图（保留原 mime，jpg 即 jpg）。不做 chroma-key、不做缩放、不拼接 style header。

**调用：**
```
python gen_with_gemini.py prompts/<asset>.txt
```

**Prompt 风格：** 中文撰写，每张 sprite sheet 一份；先在用户审阅后才调用 API。

**抠图与切片：** 由独立工具（待写）处理，**不进 gen 脚本**。

## 资产级细化设计（v2）

上一版 `dingbu_tileset_v1.jpg` 的问题不是模型完全跑偏，而是设计稿只给了"有哪些东西"，没有给出"每个东西为什么一眼能被认出来"。从 v2 开始，生成策略改为：**先补足物件级美术设定，再分表生成**，避免一张综合 8×8 表里所有 tile 都只是概念占位。

### 生成拆分策略

不再优先生成一张全量大 tileset。改成以下 5 类资产分开生成、分开验收：

| 类别 | Prompt 文件 | 目的 | 验收重点 |
|------|-------------|------|----------|
| 地表基础 | `ground_tileset_v2_zh.txt` | 草地、泥路、操场、水泥、过渡边 | 可平铺、无黑格线、颜色统一 |
| 完整建筑 | `*_building_full_*_zh.txt` | 教学楼、宿舍、小卖部、食堂、厕所 | tile 按实际顺序摆放，整张 sheet 能看出完整建筑 |
| 其它建筑模块 | `other_buildings_tileset_v1_zh.txt` | 宿舍、小卖部、食堂、厕所的补充模块 | 四类建筑能一眼区分，仍保持同一校园风格 |
| 山林边界 | `nature_border_tileset_v1_zh.txt` | 山林、岩石、土坡、边界阴影 | 能形成"山里挖平一块地"的自然围合 |
| 校园小物 | `campus_props_tileset_v1_zh.txt` | 旗杆、乒乓球台、长椅、水桶等 | 增加童年小学细节，不抢主体建筑 |
| 淘米池 | `rice_washing_pond_tileset_v1_zh.txt` | 2×2 淘米池 + 湿泥/水桶 | 不能像泳池，要像乡村水泥/石头洗米池 |

### Tile 分隔与透明背景

最终目标不是在 atlas 里保留黑色分割线，而是让每个 tile 之间有透明 gutter，便于切片和后续 Godot 导入。

由于当前 `gen_with_gemini.py` 只保存模型返回的原始 mime，实际生成曾返回 JPG，而 JPG 不支持 alpha 透明，所以不应依赖模型直接给透明 PNG。更稳的做法是：

1. Prompt 中要求：每个 tile 之间和画布外圈保留固定宽度的纯洋红 `#FF00FF` gutter。
2. 严禁黑色格线、深色边框、编号、标签。
3. 图块内容本身不得使用 `#FF00FF`。
4. 后处理脚本把 `#FF00FF` 及其 JPG 压缩近似色按容差抠成 alpha。
5. 切片时按 `rows / cols / gutter` 裁掉 gutter，只保留 tile 内容区域。

推荐默认参数：

| 用途 | 逻辑 tile | 生成 cell | gutter |
|------|-----------|-----------|--------|
| 地表 tile | 32×32 | 128×128 | 8 px |
| 小物/自然物 | 32×32 或 64×64 | 128×128 | 8 px |
| 建筑 tile | 32×32 或 64×64 | 128×128 | 8 px |

如果模型仍然画出黑线，这张图只能作为预览，不进入正式切片；正式切片版本必须重生或后处理裁掉线。

### 建筑 tile 的实际顺序

建筑不再只做"零散模块表"。每栋建筑都优先做一张完整建筑 sheet：格子数量直接对应该建筑在地图里的占地尺寸，tile 在 sheet 中的行列顺序就是它在地图里的实际摆放顺序。这样有两个好处：

1. 用户直接看生成图就能判断这栋建筑整体是否像记忆中的建筑。
2. 后处理切片后，按同样行列摆回 TileMap，就能还原完整建筑，不需要再猜模块怎么拼。

具体规格：

| 建筑 | 地图尺寸 | 完整 sheet prompt | 说明 |
|------|----------|------------------|------|
| 教学楼 | 10×3 | `teaching_building_full_10x3_v1_zh.txt` | 操场正北，学校主体 |
| 宿舍 | 7×2 | `dormitory_building_full_7x2_v1_zh.txt` | 北侧生活区，长条低矮 |
| 小卖部 | 3×2 | `shop_building_full_3x2_v1_zh.txt` | 宿舍东侧，带小门面 |
| 食堂/打热水房 | 4×2 | `canteen_building_full_4x2_v1_zh.txt` | 小卖部南侧，有烟囱/水桶 |
| 厕所 | 2×2 | `toilet_building_full_2x2_v1_zh.txt` | 主路东端北侧，低调偏旧 |

后续如果需要复用模块，再从这些完整建筑 sheet 中抽出墙面、屋顶、门窗等模块，或单独生成模块补充表。

### 视觉叙事锚点

地图需要有几个强识别点，让玩家即使不看文字也能知道这是"山里的乡村小学"：

1. **教学楼是视觉中心**：白灰墙/小瓷砖墙、成排窗户、木门、小台阶，放在操场正北。
2. **操场不是现代操场**：没有塑胶跑道，只有被踩平的沙土地、脚印、淡淡白线和草地边缘。
3. **泥巴主路是动线核心**：东西向 2 tile 宽，带车辙、脚印、碎石和干湿色差。
4. **山林没有围墙感**：边界不是一堵墙，而是树、岩石、土坡、杂草自然堆叠。
5. **生活区靠北偏右**：宿舍、小卖部、食堂、淘米池挤在一起，要有生活气息：水桶、灶烟、遮阳棚、旧木门。
6. **厕所远离主体**：小、旧、偏角落，位置在主路东端北侧，视觉上不能比教学楼抢眼。

### 地表细化

地表 tile 负责把地图从"平铺颜色块"变成可看的校园地面。每种地表都至少需要"中心 tile + 边缘/过渡 tile + 细节变体"。

| 地表 | 必备 tile | 视觉要点 |
|------|----------|----------|
| 草地 | 普通草、深草、踩踏草、草接泥边 | 黄绿偏灰，不要荧光；杂草少量、方向随机 |
| 主泥路 | 横向车辙、纵向小路、路边草、泥坑 | 赭黄色干泥，局部深色湿泥；主路宽 2 tile |
| 操场沙地 | 沙地中心、脚印、白线、沙接草边 | 浅沙黄微灰，颗粒细，不要像海滩 |
| 水泥小道 | 旧水泥、裂缝、水泥接草边 | 只少量使用，不能变成现代硬化校园 |
| 湿泥 | 水桶旁湿泥、淘米池旁泥水 | 深赭棕，边缘不规则 |
| 石板/碎石 | 不规则石块、碎石过渡 | 用于厕所附近、山脚、池边 |

### 建筑细化

建筑 tile 不追求一次生成完整建筑，而是要能拼装。每栋建筑必须有自己的"身份特征"，否则会像普通 RPG 房子。

#### 教学楼

- 形象：学校主体，朴素、规整、有人使用，不是废墟。
- 材质：白灰墙或浅赭小瓷砖墙，旧白窗框，淡蓝玻璃，木门，深灰水泥勒脚。
- 模块：屋顶/女儿墙、窗墙、纯墙、门、小台阶、底座、侧边外角。
- 细节：成排窗户要整齐；风化只在墙脚和窗台附近，不能太脏。
- 避免：欧式尖屋顶、教堂感、现代教学楼玻璃幕墙。

#### 宿舍

- 形象：比教学楼更矮、更生活化，长条房。
- 材质：灰瓦坡屋顶、白墙或浅土黄墙、小窗、窄木门。
- 细节：窗户可以更小更密，墙脚有晾晒痕迹/水渍；门口可有一小块水泥台阶。
- 避免：豪华民宿、现代公寓、欧式木屋。

#### 小卖部

- 形象：紧挨宿舍的小体量店铺。
- 材质：同校园旧墙体，但门面更杂：小窗口、木柜台、布棚/铁皮棚。
- 细节：可以有空白木牌或蓝色遮阳棚，但不要生成可读文字；门口可有木箱。
- 避免：便利店、霓虹招牌、现代玻璃门。

#### 食堂/打热水房

- 形象：实用、略旧、有烟火气。
- 材质：红砖/灰砖局部外露，灰瓦，木门或旧铁门。
- 细节：小烟囱、白色蒸汽、门口水桶、墙边水龙头、柴火/煤堆暗示。
- 避免：餐厅、厨房内景、现代不锈钢食堂。

#### 厕所

- 形象：小、偏远、低调。
- 材质：灰砖或红砖墙，灰瓦/水泥顶，单扇旧木门。
- 细节：墙脚杂草、碎石、小水泥踏步；不要画明显文字或符号。
- 避免：现代公厕、太干净、太醒目的标志。

### 山林边界细化

边界应当像山地自然压到校园边缘，而不是地图外框。tile 要能拼出 3 tile 厚的围合层：

- 第 1 层（最外）：深色密林、重叠树冠、暗部，遮住地图外侧。
- 第 2 层：树干、灌木、岩石、大块土坡，形成厚度。
- 第 3 层（内侧）：草地接山脚、碎石、低灌木，让校园草地自然过渡进去。

树和岩石比例仍按 70% 树 / 30% 岩石，但不要机械随机。角落要有更密的树和阴影；靠路口的边界可以露出泥土坡。

### 校园小物细化

小物不是越多越好，目标是让地图有"小学生活"而不破坏静态地图清晰度：

| 小物 | 放置建议 | 视觉要点 |
|------|----------|----------|
| 旗杆 | 教学楼前/操场边 | 细杆 + 水泥底座 + 小红旗，旗不要太大 |
| 乒乓球台 | 操场边缘 | 绿色水泥台，白中线，旧但完整 |
| 木长椅 | 路边/树下 | 简单两三块木板，深棕描边 |
| 水桶 | 淘米池/食堂旁 | 蓝灰或木桶，体积小 |
| 扫帚/簸箕 | 教学楼墙边 | 只能作为装饰小物，不要显眼 |
| 木公告板 | 教学楼旁 | 空白木板，不写字 |
| 电线杆/路灯 | 主路旁少量 | 水泥杆，朴素，不要现代城市路灯 |
| 碎石/杂草 | 山脚、厕所旁 | 用于过渡和破除空地感 |

### 淘米池细化

淘米池是这张地图里很有个人记忆感的物件，应该单独做，不要用通用"水池"替代。

- 尺寸：最终地图上 2×2 tiles。
- 材质：低矮水泥/石头池沿，灰白、边缘有磨损。
- 水面：蓝绿但偏灰，浅水，少量白色波纹；不能亮蓝，不能像游泳池。
- 周边：湿泥、碎石、水桶、洗米盆、脚印。
- 视角：俯视为主，池沿有轻微厚度。
- 避免：喷泉、泳池、池塘、现代瓷砖水槽。

### 下一次生成验收标准

1. 一张图不需要一次性包含所有类别；每张图先做到该类别细节足够。
2. 每个格子必须能按固定网格裁切；使用洋红 gutter 转透明，不要黑色格线，不要编号，不要说明文字。
3. 地表 tile 要优先检查四向平铺；建筑 tile 要优先检查能不能拼出目标尺寸。
4. 生成图可以是 1024×1024，但逻辑上必须明确几行几列；后处理再切片缩放。
5. 如果模型继续生成黑格线，接受作为预览，但正式切片要在后处理里裁掉格线或要求重生。

## 生成方式（地图渲染）

**工具：** Python 3 + Pillow（独立脚本，不依赖 Godot 运行时）

**脚本位置：** `tools/generate_map.py`（待写）

**前置条件：** 所有 tileset sprite sheet 已生成、抠图、切片为独立 32×32 PNG 放在 `assets/tilesets/dingbu/tiles/`（待规划具体路径）。

**流程：**

1. 加载切片好的 tile PNG 到内存
2. 创建 2048×1280 的空白画布
3. 按"地表分层"顺序填充每一层
4. 写出 `school.png`
5. 同时写出：
   - `school_tileset.tres`：Godot 4 格式的 TileSet 资源，引用本地 tile PNG
   - `school.tscn`：Godot 4 场景，含一个 Node2D 根节点 + 一个 TileMap 节点
6. tile PNG 用 `res://assets/tilesets/dingbu/tiles/xxx.png` 相对路径引用（待规划具体路径）

**确定性：** 树/岩石的随机分布使用固定随机种子（`seed=42`），保证多次运行产物完全一致。

**抠图与切片工具（待写）：** `tools/ai/postprocess.py`
- 输入：sprite sheet 原图（jpg/png）+ 网格规格（行数/列数/单 tile 像素/gutter 像素）
- 步骤：chroma-key 洋红 gutter → alpha 通道 → 按网格裁掉 gutter 并切片 → NEAREST 缩放到目标 tile 尺寸（如 32×32 或 64×64）→ 输出独立 PNG，命名按 tileset 内位置（如 `teaching_building_2_3.png`）
- JPG 容差：因为模型可能返回 JPG，洋红背景会被压缩污染，抠图不能只匹配精确 `#FF00FF`，需要按 RGB 距离或 HSV 阈值做容差。
