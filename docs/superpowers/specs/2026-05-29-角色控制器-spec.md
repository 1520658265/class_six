## 四、控制器实现

### 4.1 InputMap（写入 project.godot）

```ini
[input]

move_up={
"deadzone": 0.5,
"events": [Object(InputEventKey,"physical_keycode":87),Object(InputEventKey,"physical_keycode":4194320)]
}
move_down={
"deadzone": 0.5,
"events": [Object(InputEventKey,"physical_keycode":83),Object(InputEventKey,"physical_keycode":4194322)]
}
move_left={
"deadzone": 0.5,
"events": [Object(InputEventKey,"physical_keycode":65),Object(InputEventKey,"physical_keycode":4194319)]
}
move_right={
"deadzone": 0.5,
"events": [Object(InputEventKey,"physical_keycode":68),Object(InputEventKey,"physical_keycode":4194321)]
}
run={
"deadzone": 0.5,
"events": [Object(InputEventKey,"physical_keycode":4194325)]
}
interact={
"deadzone": 0.5,
"events": [Object(InputEventKey,"physical_keycode":90),Object(InputEventKey,"physical_keycode":4194309)]
}
```

physical_keycode 说明：
- 87/83/65/68 = W/S/A/D
- 4194319/22/20/21 = 上/下/左/右方向键（KEY_UP/DOWN/LEFT/RIGHT）
- 4194325 = Shift（KEY_SHIFT）
- 90 = Z
- 4194309 = Enter（KEY_ENTER）

`interact` 仅声明键位，不接业务（业务在对话 spec 处理）。

### 4.3 控制器脚本

```gdscript
# scripts/character/yuansheng_controller.gd
extends CharacterBody2D

const SPEED_WALK := 120.0
const SPEED_RUN := 220.0

@onready var sprite: AnimatedSprite2D = $AnimatedSprite2D
var _facing: String = "down"

func _ready() -> void:
    sprite.sprite_frames = Sprites.walk_frames(AssetIds.Char.YUANSHENG)
    sprite.animation = "walk_down"
    sprite.frame = 0
    sprite.pause()
    print("[yuansheng] controller ready, animations: ", sprite.sprite_frames.get_animation_names())

func _physics_process(_delta: float) -> void:
    var input := Input.get_vector("move_left", "move_right", "move_up", "move_down")
    var speed := SPEED_RUN if Input.is_action_pressed("run") else SPEED_WALK
    velocity = input * speed
    move_and_slide()
    _update_animation(input)

func _update_animation(input: Vector2) -> void:
    if input == Vector2.ZERO:
        sprite.pause()
        return
    var dir := _pick_direction(input)
    var anim := "walk_" + dir
    if sprite.animation != anim:
        sprite.play(anim)
    elif not sprite.is_playing():
        sprite.play()
    _facing = dir

func _pick_direction(input: Vector2) -> String:
    if abs(input.x) >= abs(input.y):
        return "right" if input.x > 0 else "left"
    return "down" if input.y > 0 else "up"

func get_facing() -> String:
    return _facing
```

`get_facing()` 留作后续对话/互动 spec 使用（"按 Z 时面前是谁"）。

### 4.4 动画方向选择规则

| 输入向量 | 主轴判定 | 选择动画 |
|---|---|---|
| `(0, 0)` | — | pause（停在当前帧） |
| `\|x\| ≥ \|y\| && x > 0` | 横轴 | walk_right |
| `\|x\| ≥ \|y\| && x < 0` | 横轴 | walk_left |
| `\|x\| < \|y\| && y > 0` | 纵轴 | walk_down |
| `\|x\| < \|y\| && y < 0` | 纵轴 | walk_up |

斜向输入时 `\|x\| == \|y\|` 落入横轴分支（取 right/left），即"45° 走优先表现为水平"。这是俯视像素 RPG 的常见约定。

### 5.2 验收清单

打开 Godot 编辑器按 F5：

1. 主角出现在屏幕中央，处于 walk_down 第 0 帧静止
2. 按 W/S/A/D 或方向键 → 主角移动且动画切换正确（4 方向）
3. 松开输入 → 动画停在当前帧
4. 按住 Shift + 移动 → 速度明显加快
5. 走到任一墙边 → 不穿过
6. 从 BlockerCenter 北侧（上方）走过 → 主角在 blocker 之上
7. 从 BlockerCenter 南侧（下方）走过 → blocker 在主角之上（y_sort 验证）
8. Camera2D 平滑跟随主角中心

Headless 烟雾验证：

```powershell
.\tools\godot.cmd --headless --path . --quit-after 60
```

期望：退出码 0，stdout 出现 `[yuansheng] controller ready, animations: ["walk_down", "walk_left", "walk_right", "walk_up"]`，无 SCRIPT ERROR。

## 六、范围与排除

本 spec **包含**：

- 控制器脚本 + 4 方向动画切换 + Shift 快跑
- yuansheng.tscn 独立场景（含 CharacterBody2D / AnimatedSprite2D / CollisionShape2D / Camera2D）
- main.tscn demo room（4 墙 + 1 中心 blocker，无 NPC）
- InputMap 6 个 action（move_up/down/left/right + run + interact）
- 顶层文档视角注解同步

本 spec **不包含**（各自后续 spec）：

- NPC 节点结构与 spawn
- 对话系统（interact 键的业务）
- 场景过渡 / 地图门
- 战斗系统（横版/俯视战斗模式后续讨论）
- UI（HP/属性面板/小卖部）
- 物理细节微调（推箱子、滑冰地板等）
- 角色冬装/春夏装切换（后续多角色 controller 共用 base 时再抽）

## 七、后续扩展预留

- **多角色复用**：将控制器拆为 `BaseCharacterController`（移动+动画切换）+ `YuanshengController`（具体角色）。本 spec 不抽，加第二个角色时再抽（YAGNI）。
- **状态机升级**：本 spec 用 if/match。当状态超过 5 个（如加入 push/被推/坐姿/趴下）时再升级到 State Pattern。
- **季节服装切换**：未来 yuansheng 冬装/春夏装通过 `Sprites.walk_frames("yuansheng_winter")` 切换（manifest 加新条目即可），控制器逻辑不变。
