## 一、四项关键决策

| 决策项 | 选择 | 影响 |
|---|---|---|
| Demo 切片 | 序章 + 第一章(纯叙事,无战斗) | 不准备战斗相关一切资产 |
| 主角 sprite 规格 | 32×32 像素 | 全部角色像素图按此规格制作 |
| 语言路线 | 纯普通话文本,无配音 | 不做配音,文案全部书面普通话 |
| 命名策略 | 地名真,人物化名 | 金寨/古碑/丁埠保留;同学和老师全部化名 |

## 六、工具链与工程规范

### 6.1 工具

- **引擎**:Godot 4.3 LTS(demo 期不升级)
- **像素画**:Aseprite(收费,行业标准) 或 LibreSprite(免费替代)
- **音频**:Audacity / Reaper

### 6.2 项目目录约定

```
scenes/
scripts/
assets/
  art/
    characters/
    scenes/
    ui/
  audio/
    bgm/
    sfx/
    ambient/
data/   (对白表 / 数据表 CSV/JSON)
docs/   (本文档及后续设计文档)
```

### 6.3 命名规范

- **资产文件名走英文**(避免跨平台路径坑,保证 Godot import 稳定)
- **文档保持中文**(契合作者偏好,且文档不进 import 流水线)
- **生成用 prompt 文件走中文**，并按资产类型分类，方便和美术清单逐项对应；prompt 不直接进入 Godot import 流程
- 资产 ID 命名示例:`char_yuansheng_idle.png` / `bgm_school_daily.ogg`

Prompt 目录示例:

```text
tools/ai/prompts/角色精灵/主角元生_行走表_v1.txt
tools/ai/prompts/场景建筑/教学楼_完整10x3_v1.txt
tools/ai/prompts/地表自然/地表道路_tileset_v2.txt
tools/ai/prompts/校园道具/校园杂物_tileset_v1.txt
```

### 6.4 版本控制

- 已是 git 仓库,沿用现有
- git LFS:demo 量级不需要,后续音频量大时再启用

## 七、流程与里程碑

### 7.1 资产清单表

每张图 / 每段音乐分配一个 ID,跟踪状态:未做 / 草图 / 完成。

### 7.2 四个里程碑

| 里程碑 | 交付物 |
|---|---|
| **M1** | 美术规格 + 对白脚本表 + 数据表全部定稿 |

### 7.3 测试计划

- 自测 + 2-3 位朋友试玩
- 反馈表测三个维度:节奏 / 操作 / 共鸣度

## 九、Demo 阶段不做的事(显式排除)

为避免范围蔓延,以下内容明确**不在本 Demo 准备范围**:

- 战斗系统所有美术(技能图标、HP / 勇气槽 UI、敌人 AI 数据等)
- 战斗 BGM 与战斗音效
- 对白配音
- 红色历史相关美术
- 时代品牌相关美术
- 后山板栗、采茶、生炉子等小游戏资产
- 多结局结算界面

## 十、Demo 启动前必须先定下的 3 件事

清单中所有项目都依赖这三个先决条件,**任意一项未定都无法起稿**:

1. **化名映射表**(决定文案表怎么写、NPC 表怎么填)
2. **对白脚本表的格式模板**(决定后续所有文案的填写规范)
3. **资产命名规范的具体例子**(决定美术 / 音频文件如何命名)

这三项建议在进入 M1 里程碑前完成。
