> 验图工具：在 `assets/art/` 目录下找原始 `_v<n>` 文件（jpg 或带洋红背景的 png）做对比；切片 sheet 可以用 `python tools/ai/check_grid.py <png> <rows> <cols>` 加叠绿网格。
> 重要前提：AI 没法读 PNG/JPG 像素内容，下面的"可疑"判断**只是基于 chroma_key 的统计输出**（cand/core/halo/matte/grid 像素数），不是看图得出的结论。最终对错由你来判。

---

## 字段含义

每行一次 chroma_key 调用打印的统计：

- **cand**：图里所有"R 和 B 都比 G 高"的洋红族候选像素数
- **core**：被判定为背景、立刻设透明的像素
- **halo**：背景外圈的弱洋红光环像素，被吸收为透明
- **matte**：被识别为"白色/浅灰大块矩形垫底" 而清掉的像素
- **grid**：被识别为"低饱和等距格线"而清掉的像素

判读规则：

- `cand=0` 或 `cand` 极小 → 图原本就**没有洋红背景**（可能是写实场景图），脚本基本没干活。看是不是误投到了批处理里
- `core` 远小于 `cand` → 部分洋红没被识别为背景，可能仍残留洋红块
- `matte` 很大 → 脚本把图里大块浅色区域当矩形垫底删掉了，**可能误删画面内容**
- 输出 alpha 100% 透明 → 整张被吃光，多半是 chroma_key 失败

## 排查工作流建议

1. **A 类先决定**：场景图到底要不要 alpha？如果是当背景全屏铺，其实保持不透明就行，那这几张原图无须做任何处理（直接用 `_v1_source.png`）；如果做完整透明背景图用错了角度，需要单独重生成。

2. **B 类直接看 hit_lines 原图**：
   ```powershell
   python tools/ai/check_grid.py "assets/art/characters/yuansheng/battle/effects/char_yuansheng_fx_hit_lines_v1.png" 1 1
   ```
   如果原图就是空的，那是上游生成失败，不是 chroma_key 的锅。

3. **C 类 + D 类合在一起逐张目检**。打开输出 PNG 对比同目录下原始 `_v<n>` 文件。常见问题：
   - 角色身上的白色衣服被 matte 清除掉了（出现"窟窿"）
   - 浅黄/浅灰背景物件被当垫底删了
   - 残留的粉红块（洋红没清干净）

4. **需要重跑某张图**时，参考调整：
   - 误删浅色块：加 `--matte-cleanup off`
   - 残留洋红：加 `--tolerance 8`（默认 15）或 `--mode global`
   - 边缘啃缺：加 `--halo-passes 0`
   - 例：
     ```powershell
     python tools/ai/jpg_to_png_alpha.py "assets/art/characters/jiejie/raw/char_jiejie_walksheet_v1.jpg" -o "assets/art/characters/jiejie/raw/char_jiejie_walksheet.png" --matte-cleanup off
     ```

5. **目检 OK 的图**，可以从这份清单里删掉对应行（或挂个勾），保留待排查的部分。
