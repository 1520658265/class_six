## 目标

为丁埠小学地图项目生成主角和核心 NPC 的 sprite sheet，作为后续 Godot 场景中可用的人物素材。

**本次范围内：**
- 1 个主角 + 3 个核心 NPC，共 4 个角色
- 每个角色一张 4 行 × 4 列 = 16 帧 sprite sheet
- 每帧 32×32 像素，4 方向（下/左/右/上）× idle + 3 步行帧
- 通过 Gemini 模型生成 sprite sheet，洋红 gutter，下游切片

**不在本次范围内：**
- 角色移动逻辑、碰撞、相机
- `AnimationPlayer` / `AnimatedSprite2D` 节点配置
- 对话系统、剧情、AI 行为
- 室内造型变体（如教室坐姿、操场跑步姿）

## 时代背景

主角是 1996 年生，**2008 年（小学六年级）**作为本作时间锚点。地图美术风格也据此从原计划"1990 年代怀旧褪色调"调整为"2008 年前后中国农村小学怀旧调"——保留褪色质感，但允许出现塑料彩色小物（情侣手环、塑料水杯等）、杀马特风潮等 2000s 末期特征。

## 通用 Sprite Sheet 规范（4 角色统一）

每个角色一张 4 行 × 4 列 = 16 帧的 sprite sheet。

### 行（从上到下）= 朝向

| 行 | 朝向 |
|---|------|
| 1 | 朝下（面对镜头） |
| 2 | 朝左 |
| 3 | 朝右 |
| 4 | 朝上（背对镜头） |

### 列（从左到右）= 帧序

| 列 | 帧 |
|---|------|
| 1 | idle 站立 |
| 2 | 步行循环 1（左脚向前） |
| 3 | 步行循环 2（双脚并立中间帧） |
| 4 | 步行循环 3（右脚向前） |

### 像素规格

- 单帧逻辑尺寸：**32×32**（与地图 tile 同尺寸）
- 生成时单 cell 放大到 128×128 让模型有发挥空间
- cell 之间和外圈预留 **8 px 洋红 `#FF00FF` gutter**，便于 chroma-key 抠图
- sheet 总逻辑尺寸：(128×4 + 8×5) = 552×552（模型实际输出约 1024×1024，下游 postprocess 切片再缩放）

### 视觉规则

- chunky pixel + 1px 深炭灰 `#2a2520` 主描边
- 角色脚底贴 cell 下沿（站在 tile 上的视觉感）
- 主角约 3.5-3.8 头身（头明显偏大、比例紧凑，保留小学六年级年龄感）；老师可调到 4.5 头身
- 光源统一左上 45°
- 调色板与地图 tileset 一致，褪色为主，但允许鲜艳点缀（红领巾、塑料手环等）
- 同一角色 16 帧间发型/服装/配件/比例严格一致；只有腿和摆动手在步行帧间变化

### 步行动画通用约束

- 头部和躯干在 16 帧间保持稳定，最多 1 像素整体上下浮动
- 朝下/朝上方向：双腿前后交替（看到鞋底/鞋背）
- 朝左/朝右方向：双腿前后错开
- idle 帧（列 1）= 双脚并立、自然站姿

## 角色与地图的尺寸关系

- 地图 tile 32×32，角色单帧也是 32×32
- 角色脚底贴 cell 下沿，因此角色"占地"约 1×1 tile
- 视觉上头部会突出到上方相邻 tile
- Godot 中需通过 `y_sort` 按角色脚底 y 坐标做深度排序，使角色能被前景建筑/树木遮挡

## 角色生成命令

```
python tools/ai/gen_with_gemini.py "tools/ai/prompts/角色精灵/主角元生_行走表_v1.txt"
python tools/ai/gen_with_gemini.py "tools/ai/prompts/角色精灵/班主任鲍先进_概念_v1.txt"
python tools/ai/gen_with_gemini.py "tools/ai/prompts/角色精灵/男同学模板_概念_v1.txt"
python tools/ai/gen_with_gemini.py "tools/ai/prompts/角色精灵/女同学模板_概念_v1.txt"
```

每张 sheet 生成后由 `postprocess.py`（待写）抠图 + 切片为 16 个 32×32 PNG，存放：

```
assets/art/characters/<character_name>/walk/<direction>_<frame>.png
例：assets/art/characters/protagonist/walk/down_idle.png
   assets/art/characters/protagonist/walk/down_walk1.png
   assets/art/characters/protagonist/walk/left_idle.png
   ...
```

文件命名约定：`<direction>` ∈ {`down`, `left`, `right`, `up`}，`<frame>` ∈ {`idle`, `walk1`, `walk2`, `walk3`}。

## 验收标准

1. 16 帧间角色身份特征严格一致（发型/服装/配件不漂移）
2. 主角右手在所有 16 帧都插兜，姿势不变
3. 红领巾在 4 个方向都可见（朝上方向也能看到绕过脖子的两端）
4. 步行动画跨方向一致：4 个方向的步行节奏匹配，便于循环播放
5. 不出现智能手表、电子屏幕、现代品牌 logo、英文字母
6. 抠图后无洋红残留、无 JPG 压缩污染色块
7. 16 个独立 PNG 切片后，每张严格 32×32，背景透明
