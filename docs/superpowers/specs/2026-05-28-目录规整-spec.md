日期：2026-05-28
状态：待评审
关联：所有现有 specs 与文档（迁移会动到它们的位置/命名）

## 目标

把 `D:\PersonalProject\class_six` 整理成一个职责清晰、可维护的结构，区分四条主线：

- 美术工程产出：`assets/`
- 生成工具与中间产物：`tools/`
- 设计与参考文档：`docs/`
- 工作日志：`docs/journal/`

## 原则

- **职责单一**：一个目录只装一类东西。
- **第三方与自产分离**：`assets/tilesets/{kenney,opengameart}` 是第三方下载，独立成域；自产实验产物收进 `tools/ai/_experiments/`。
- **过渡期可读**：旧英文 prompt 不删，归档到 `_archive/`，README 注明用途。
- **YAGNI**：不为"将来可能有的功能"留占位目录。
- **命名一致**：设计/规格类文档统一带 `-spec.md` 后缀。

## 风险与对策

| 风险 | 对策 |
|---|---|
| 移动失误把文件丢到错位置 | 用 `mcp__filesystem__move_file`（同名冲突会报错而不是覆盖），每批次结束 `list_directory` 核对 |
| 文档残留旧路径引用 | 批次 H 之前 `Grep` 全仓搜 `school_map/`、`memory/2026-`、`_downloads`、`-design.md`、`village_grass_cliff` 等关键词，按结果改 |
| `out/` 分类漏分 | 按映射表前缀清单一一对照；分完后 `list_directory tools/ai/out/`，根下应 0 文件、5 子目录、合计 58 个文件 |
| `_archive/` 漏归档或误归档 | 结束后 `list_directory tools/ai/prompts/`，根下应只剩 `README.md` + 8 个中文分类目录 + `_archive/`，无 .txt |
| 删 `_downloads/` 不可逆 | 用户已确认；删除前再核对一次目录内容确实只是 zip + 下载脚本/csv |

## 回滚预案

每个批次都是文件级 move，迁移路径全部记录在本文档。任何一步出错都可按映射表反向 move。全程使用 `move_file`（rename，原子操作），不做覆盖、不做"复制+删原件"。
